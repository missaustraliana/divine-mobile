# Home Dropdown Followed Lists Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Remove `New` from the Home feed dropdown and add one selectable dropdown row for each followed/subscribed curated list.

**Architecture:** Keep the existing `UI -> BLoC -> Repository` flow. `VideoFeedBlocState` will expose a typed feed source that can represent `For you`, `Following`, or one selected `CuratedList`; the UI renders dropdown rows from BLoC state and dispatches source changes. Plain `Following` fetches followed creators only; selected lists fetch only that list's ordered video refs.

**Tech Stack:** Flutter, BLoC, Riverpod provider bridge, `curated_list_repository`, `videos_repository`, Flutter l10n ARB generation, widget and BLoC tests.

## Coordination Contract

Workers must implement these public shapes so slices compose:

```dart
enum FeedMode {
  forYou,
  following,
  latest, // legacy restore value only; not selectable from Home.
}

enum VideoFeedSourceType { forYou, following, subscribedList }

final class VideoFeedSource extends Equatable {
  const VideoFeedSource.forYou();
  const VideoFeedSource.following();
  const VideoFeedSource.subscribedList({
    required String listId,
    required String listName,
  });

  VideoFeedSourceType get type;
  String? get listId;
  String? get listName;
  FeedMode get mode; // forYou for forYou, following for following/list
  String get labelFallback; // listName for list, enum name for core modes
  String get persistenceValue; // forYou, following, or list:<listId>
}
```

`VideoFeedBlocState` keeps backward-compatible construction with `mode:` and adds:

```dart
final VideoFeedSource source;
final List<CuratedList> subscribedLists;

FeedMode get mode => source.mode;
String get feedContextTitle;
bool get isSubscribedListSelected;
```

`VideoFeedSourceChanged(VideoFeedSource source)` is the new event used by the dropdown. Existing `VideoFeedModeChanged(FeedMode mode)` remains as compatibility glue and forwards to the equivalent source.

- [ ] **Step 3: Implement source model and BLoC behavior**

Implement the coordination contract. Key behavior:

- Saved `latest` restores as `VideoFeedSource.forYou()` and writes `forYou` back to SharedPreferences.
- Saved `list:<id>` restores as a subscribed-list source when `CuratedListRepository.getListById(id)` is available; otherwise restore `For you`.
- `VideoFeedSourceChanged` persists `source.persistenceValue`.
- `Following` calls `getHomeFeedVideos(authors: followingPubkeys, videoRefs: const {})`.
- Subscribed-list source calls `getVideosForList(getOrderedVideoIds(listId))` and wraps the videos in `HomeFeedResult`.
- Subscribed-list source sets `hasMore` to `false`.
- `VideoFeedCuratedListsChanged` carries `List<CuratedList>` and updates `state.subscribedLists`.
- Curated list changes refresh only when a subscribed-list source is selected; they do not refresh plain `Following`.
